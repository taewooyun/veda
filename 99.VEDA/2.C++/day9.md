## C++ 표준 입출력

- 네임스페이스란? 변수나 함수가 속해 있는 소속을 지정해 주는 역할
- 내부 식별자(형식, 함수, 변수 등) 범위를 부여, 라이브러리간 이름 충돌 방지
- std::는 std라는 네임스페이스에 접근하는 표현
    - std(standard)는 많이 사용하는 함수와 클래스 객체 유틸리티가 정의된 네임스페이스
- using namespace std; 를 사용하면 std:: 를 생략할 수 있음
    - std::뿐만 아니라 다른 네임스페이스도 생략할 수 있음
    - 네임스페이스 생략은 코드를 간단하게 하지만 생략된 네임스페이스로 범위를 착각할 수 있음
    - 또한, 컴파일러에게 혼란을 줄 수 있기 때문에 상황을 잘 고려해서 사용해야함

### cout 스트림 객체 사용

- 출력 자료형 지정 없이 정수, 부동 소수점 등을 출력 할 수 있음
- ‘<<’ 연산자를 사용하여 출력 대상을 cout 스트림 객체로 전달, 여러개 변수 전달 가능
- 개행문자는 std::endl을 cout 스트림 객체에 전달

### cin 스트림 객체 사용

- 입력 자료형 지정 없이 정수, 부동 소수점 등을 입력 할 수 있음
- ‘>>’ 연산자를 사용하여 cin 스트림 객체로 부터 변수에 값을 전달, 여러 개의 변수 입력 가능

### 와이드 문자

- 와이드 문자를 저장하는 자료형 wchar_t
- 1byte로 표현이 불가능한 언어를 와이드 문자라고 지칭함
    - 인코딩 방식에 따라 2byte char를 사용하는 멀티바이트(MBCS), wchar_t를 사용하는 유니코드(UNICODE)로 구별 됨
- wchar_t 크기 비주얼 스튜디오 2byte로 정의, GCC 컴파일러 4byte로 정의
    - gcc 컴파일 옵션 -fshort-wchar 적용으로 wchar_t를 2byte로 사용 가능

## 변수의 유효 범위와 형식 변환

### 변수의 유효 범위

- 변수가 선언된 위치에 따라 사용할 수 있는 범위가 결정
- 선언 범위가 다른 경우 변수 사용이 허용되지 않음
    - 동일 범위내 이름이 같은 변수를 사용할 수 없는 ‘단일 정의 규칙’
    - 다른 범위에서는 같은 이름으로 변수를 선언해도 오류가 발생하지 않음

### c++ 언어의 대표적인 범위

- 구문 범위 : if, for, while 등의 구문 내부
- 지역 범위 : 매개변수 이름을 포함 함수안에 선언한 이름은 해당 함수 내에서만 유효
- 전역 범위네임스페이스나 클래스, 함수 등에 속하지 않고 외부, 선언 지점부터 파일 끝까지 해당
- 클래스 범위 : 클래스 정의 해당, 클래스 멤버에 대한 접근성은 접근 지정자로 제어할 수 있음
- 네임스페이스 범위 : 네임스페이스 내부, 네임스페이스는 서로 다른 파일들의 여러 블록에서 선언될 수 있음

### 형식 변환

- 변수 선언 시 지정된 데이터 ‘형식변경’이 필요 할 경우가 존재
- 데이터 유지, 데이터 유실
    - 적은 범위 표현이 가능한 형식을 큰 범위로 변환 시 데이터 유실 없음
    - 큰 범위 표현이 가능한 형식을 작은 범위 형식으로 변경 시 데이터 유실 가능
- 표현 데이터 계열이 다를 경우 변환 불가 또는 잘못된 변환 발생
- 변환 방식에 따른 구분
    - 암시적 형변환 : 컴파일러가 자동으로 변경
    - 명시적 형변환 : 개발자가 의도적으로 직접 변경
        - float(), (float)() 등 데이터 형식을 사용한 명시적 형 변환
        - static_cast, const_cast, dynamic_cast, reinterpret_cast등 다양한 명시적 형 변환 지원
