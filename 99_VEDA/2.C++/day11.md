## 객체지향 프로그래밍

### 대표 객체와 파생 객체

- 비슷한 역할을 하는 객체의 공통 특성만 포함한 대표 객체
- 대표 객체의 공통 특성을 물려 받고 새로운 특성을 추가한 한 파생 객체
    - 상속 : 대표(상위) 객체에서 파생 객체를 만드는 것
    - 가솔린 엔진 is-a 엔진
- is-a 관계는 상호 교환이 가능함. 즉, 가솔린 엔진은 엔진을 대체 할 수 있음(has-a관계는 불가능)

### 객체 간에 접근 제어

- 직접 사용하지 않는 이벤트 사용 관계
    - 객체의 동작을 직접 제어 하지 않고 특정 사건(이벤트)로 제어하는 방법
    - 출력 조절기가 피스톤을 직접 움직이지 않고 출력을 조절하여 조절
- 접근이 금지된 객체에 접근할 수 없어야 함
    - 가속 페달이 출력 조절기가 아닌 피스톤을 직접 제어할 수 없음
    - 동력 전달 장치에 전달되는 동력은 엔진이 전달함. 가속 페달이 관여하면 안됨

### 객체지향 프로그래밍 특징

- 추상화
    - 집단의 불필요한 요소는 배제하고 공통된 특징만 추출하는 것
    - 4가지 종류의 엔진을 구동 방식으로 분류, 엔진과 내연기관 엔진이라는 추상화된 대표 객체 생성
        - 다솔린, 디젤, 가스엔진은 모두 연료를 연소시켜 동력을 얻는 **내연기관**
        - 내연기간 엔진과 전기 엔진은 차량을 구동하는 **엔진**
- 캡슐화
    - 복잡한 내부 기능을 묶어 단순화하여 동작하도록 함으로써 외부에서 불필요한 정보를 감추는 것
    - 외부에 노출될 필요가 없는 정보들을 은닉하고 해당 정보들을 사용할 수 있는 인터페이스를 제공
- 상속성
    - 부모 객체의 특성을 이어 받아 새로운 객체를 만드는 것
    - 상속 받은 특성 이외에 상속 받은 객체에 고유의 특성을 추가 하는 것이 필요함
- 다형성
    - 부모 객체의 특성을 모두 가지고 있는 자식 객체는 부모 객체 대체 가능
    - 부모 객체를 대체한 자식은 자신만의 특성을 반영하여 자식 객체 마다 다르게 동작함 → 다형성

## 클래스와 인스턴스

### 클래스

- 데이터와 함수를 포함하는 논리 단위
    - 객체, 클래스 = 변수, 자료형
- 객체가 포함하는 데이터와 함수를 정의하는 문법적인 요소
    - 데이터와 함수를 클래스 정의에 포함하며 멤버 변수, 멤버 함수로 호칭함
    - 멤버 변수, 멤버 함수는 접근을 통제하기위해 public, private과 같은 접근지정자를 사용
    - 생성자와 소멸자 : 객체가 생성될 때와 소멸될 때 자동으로 호출되는 특수 함수
- 객체 : 특정 클래스를 사용할 수 있도록 만든 변수

## 추상화와 캡슐화

### 캡슐화

- 변수와 함수를 클래스로 감싸서 외부에서 직접 접근 할 수 없도록 은닉화
- 하나의 기능이 여러 함수로 이루어질 경우 대표 함수만 노출
    - 클래스 내부의 세부 로직을 알 필요가 없음 → 클래스 변경이 외부 로직에 영향을 주지 않음

### 접근 지정자

- private : 클래스 내의 멤버 함수에서만 접근 가능
- public : 다른 클래스 어디서든 접근 가능
- protected : 클래스 내의 멤버 함수, 상속 관계의 클래스에서 접근 가능
