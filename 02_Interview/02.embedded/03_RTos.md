### RTos를 사용하는 이유

- RTOS는 bare-metal 방식보다 구조적으로 여러 작업을 Task 단위로 관리할 수 있는 실시간 운영체제입니다.
- RTOS는 범용 OS와 달리 처리량이나 사용자 편의성보다 작업 실행 시점의 예측 가능성을 우선하는 운영체제입니다.

우선순위 기반 선점형 스케줄링과 컨텍스트 스위칭, Tick 기반 시간 관리를 통해 실행 시간의 결정성을 확보할 수 있어, 하드웨어 제어와 같은 실시간성이 중요한 시스템에 적합합니다.

#### RTOS는 멀티코어가 아니다
병렬 실행은 아니지만 빠른 Context Switch로 동시 실행처럼 동작하게 함

### RTOS에서 로그 찍는법
임베디드 환경에는 stdout 개념이 없어서 newlib의 _write()를 직접 구현해 UART 같은 하드웨어로 리타겟해야 합니다. printf는 초기 디버깅엔 유용하지만, 리소스와 실시간성 문제 때문에 보통은 로그 매크로나 전용 로그 task 구조로 대체합니다.


### RTOS에서 printf를 찍으면 위험한 이유
내부적으로 `HAL_UART_Transmit()` 실행 : Task를 아무데나 탄다
thread-safe 아님
ISR에서 쓰면 블로킹 함수 호출
