좋다, 이건 **“RTOS를 아는 수준” → “실무에서 쓸 줄 아는 수준”**으로 가는 정리다.
아래 로드맵은 **실제 현업(자동차·로봇·가전·산업기기) 기준**으로
👉 *왜 쓰는지 / 어디서 터지는지 / 면접에서 말할 수 있는지* 기준으로 짰어.

# 🧭 RTOS 실무 학습 로드맵 (요소별 인덱스)

> 기준 RTOS: **FreeRTOS (STM32Cube + CMSIS-RTOS v2)**

---

## 0️⃣ RTOS 기본 인식 단계 (필수 베이스)

### 🎯 목표

* “왜 RTOS를 쓰는지”를 **논리적으로 설명 가능**

### 학습 요소

* RTOS vs Bare-metal 차이
* Preemptive Scheduling
* Context Switch 개념
* Tick 기반 시간 관리

### 실무 포인트

* **RTOS는 멀티코어가 아니다**
* “동시성”과 “병렬성” 구분

---

## 1️⃣ Task 설계 & 스케줄링 (⭐⭐⭐⭐⭐)

### 🎯 목표

* Task를 **기능 단위로 쪼개는 능력**

### 학습 요소

* Task 생성 / 삭제
* Priority 설계
* `vTaskDelay` vs `vTaskDelayUntil`
* Rate Monotonic Scheduling

### 실무 예

* Control Task (10ms)
* Communication Task (Event-driven)
* Logging Task (Low priority)

### 체크포인트

* 데드라인 기준 우선순위 설명 가능?

---

## 2️⃣ 시간 관리 & Deadline 제어 (⭐⭐⭐⭐)

### 🎯 목표

* “주기 보장 + Miss 감지” 가능

### 학습 요소

* Tick vs Real Time
* Deadline miss 감지
* Timeout 기반 설계
* Watchdog 연계

### 실무 예

* 센서 샘플링 주기 보장
* 응답 시간 제한 통신

---

## 3️⃣ Task 간 통신 (IPC) (⭐⭐⭐⭐⭐)

### 🎯 목표

* **전역 변수 없이 구조 짜기**

### 학습 요소

* Queue
* Stream / Message Buffer
* Event Group

### 실무 예

* ISR → Task 데이터 전달
* Producer / Consumer 구조

### 체크포인트

* Queue vs Semaphore 차이 설명 가능?

---

## 4️⃣ 동기화 & 자원 보호 (⭐⭐⭐⭐⭐)

### 🎯 목표

* **Race Condition 제거 능력**

### 학습 요소

* Binary Semaphore
* Mutex
* Priority Inheritance

### 실무 예

* UART / I2C 공유
* 로그 출력 보호

### 실무 사고 포인트

* Priority Inversion 시나리오 설명 가능?

---

## 5️⃣ ISR ↔ RTOS 연동 (⭐⭐⭐⭐)

### 🎯 목표

* 인터럽트와 Task의 경계 이해

### 학습 요소

* FromISR API
* Deferred Interrupt Processing
* ISR 최소화 원칙

### 실무 예

* 버튼 ISR → Event Task
* DMA 완료 ISR → 처리 Task

---

## 6️⃣ 메모리 관리 & Stack 설계 (⭐⭐⭐⭐)

### 🎯 목표

* **Stack Overflow 안 터뜨리기**

* Dynamic vs Static allocation

### 실무 포인트

## 7️⃣ 소프트웨어 타이머 & 이벤트 기반 설계 (⭐⭐⭐)

# 📌 실무자 기준 “완주 기준”

### 이걸 할 수 있으면 **RTOS 실무 투입 가능**

* [ ] Task 주기/우선순위 설계 가능
* [ ] Queue / Mutex 설계 이유 설명 가능
* [ ] ISR에서 직접 처리 안 하는 이유 설명 가능
* [ ] Deadline miss 대응 전략 있음
* [ ] Stack/Heap 문제 디버깅 경험 있음

# 🎯 RTOS 포트폴리오 프로젝트 주제 모음

## 🟢 Level 1 — RTOS 기본기 증명 (입문 ~ 필수)

### 1️⃣ 멀티 태스크 센서 모니터링 시스템

**(가장 무난 + 면접관 이해도 최고)**

**구성**

* Sensor Task (주기 10ms)
* Display / UART Task (100ms)
* Logging Task (Low priority)
* Queue 기반 데이터 전달

**RTOS 포인트**

* `vTaskDelayUntil`
* Priority 설계
* Queue (Producer / Consumer)

**면접에서 말할 수 있는 것**

> “Bare-metal로도 가능하지만, 주기 보장과 구조 분리를 위해 RTOS를 사용했습니다.”

---

### 2️⃣ 버튼 이벤트 기반 상태 머신 시스템

**(ISR ↔ Task 개념 증명용)**

**구성**

* Button ISR → Event Task
* State Task (IDLE / RUN / ERROR)
* LED / UART 출력

**RTOS 포인트**

* `xQueueSendFromISR`
* Event-driven Task
* ISR 최소화 설계

---

## 🟡 Level 2 — 실무 감각 어필 (⭐⭐⭐ 중요)

### 3️⃣ 주기 제어 + 통신 분리 시스템 (추천 ⭐⭐⭐⭐⭐)

![Image](https://freertos.org/media/2018/tskstate.gif)

![Image](https://cdn.shopify.com/s/files/1/0559/1970/6265/files/functionality_of_RTOS_480x480.jpg?v=1652780045)

![Image](https://www.digikey.com/maker-media/b191bc2f-290a-44be-8ce0-6e1d4cdfd04d)

**구성**

* Control Task (10ms, High)
* Comm Task (Event-driven)
* Log Task (Low)
* UART Mutex

**RTOS 포인트**

* Deadline 기반 Task
* Mutex + Priority Inheritance
* Control/Comm 분리 설계

**면접 핵심 멘트**

> “제어 Task가 통신 지연의 영향을 받지 않도록 분리했습니다.”

---

### 4️⃣ ISR 기반 DMA 통신 처리 시스템

**(실무 냄새 강함)**

**구성**

* UART DMA RX ISR
* Packet Parse Task
* Response Task

**RTOS 포인트**

* Deferred Interrupt Processing
* Queue / Stream Buffer
* Timeout 기반 처리

**어필 포인트**

* “ISR에서 파싱 안 한 이유 설명 가능”

---

## 🟠 Level 3 — 한 단계 위 (차별화)

### 5️⃣ Deadline Miss 감지 & 복구 시스템

**(RTOS 이해도 최상급)**

**구성**

* Periodic Task (10ms)
* Execution Time 측정
* Deadline Miss 발생 시 상태 전환

**RTOS 포인트**

* `vTaskDelayUntil`
* Tick / Cycle Counter
* Watchdog 연계

**면접에서 강력**

> “Deadline miss를 로그로만 남기지 않고 시스템 상태로 반영했습니다.”

---

### 6️⃣ 우선순위 역전 시뮬레이션 프로젝트

**(면접관 놀람 확정)**

**구성**

* Low / Medium / High Task
* 공유 자원 (UART)
* Mutex vs Semaphore 비교

**RTOS 포인트**

* Priority Inversion
* Priority Inheritance 실험

---

## 🔵 Level 4 — 고급 & 실무형 (선택)

### 7️⃣ 저전력 RTOS 시스템

**(가전·IoT 지원 시 유리)**

**구성**

* Event-driven Task 위주
* Idle Hook
* Tickless Idle

**RTOS 포인트**

* Power + RTOS 통합 설계

---

### 8️⃣ RTOS 기반 미니 프레임워크

**(소프트웨어 설계력 어필)**

**구성**

* Task 생성 매니저
* Message API 래퍼
* RTOS 의존성 최소화

**어필 포인트**

* “RTOS 교체 가능 구조”
